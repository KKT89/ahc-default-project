# Codex 向け指示

## このリポジトリについて

- すべての回答は日本語で記載すること。
- 問題文は `task.html` に保存されている。

## コーディングスタイル

- C++ コードでは `for` 文の直後に来る文でも `{}` を省略しないこと。

## パラメータ管理

- `main` 関数内に数値定数をハードコーディングしないこと。チューニング対象のパラメータは `lib/hp_params.hpp` の `HP_PARAM` マクロで宣言し、Optuna の探索対象とすること。
- ただし、処理の分岐を制御する離散値（例: `0`, `1`, `2` で計算式を切り替えるフラグ）は定数として直接記述してよい。この種の値は `HP_PARAM` で宣言せず、Optuna の探索対象に含めないこと。
